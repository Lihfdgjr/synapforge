# 3D Understanding — Cheap Path

Goal: native 3D world understanding for our 375M LNN+SNN, with **140 GPU-h
(≈¥980)** total budget and ≤80GB free data.

## Recipe: DUSt3R-as-data-engine + Plücker rays + EGNN equivariant adapter

### Why this combination

- **DUSt3R / MASt3R** (Wang 2312.14132, 2406.09756) takes 2 photos and outputs
  a per-pixel 3D pointmap, feedforward, no scene-fitting. Run **frozen** as a
  teacher → infinite pseudo-labels from any 2-photo source.
- **Plücker ray embedding** per pixel (6-dim per ray, MLP→256-dim) encodes camera
  geometry as input tokens; LFN trick (Sitzmann) but as conditioning, not output.
- **EGNN equivariant adapter** (Satorras 2102.09844) on point_cloud branch only —
  rotation/translation invariance for free, ~10× data efficiency.
- **CfC + PLIF base stays unchanged** — we extend, not replace.

### Data (free, 80GB total)

| Source | Volume | Cost | Use |
|--------|--------|------|-----|
| Habitat-Sim (HM3D + Gibson) | 20K trajectories × 2-3 RGB views @ 256² | 5GB compute, 12 CPU-h render | Indoor scene grounding |
| Objaverse-LVIS subset | 50K filtered assets × 4 views = 200K pairs | 30GB | Object diversity |
| ImageNet-1K + DUSt3R pseudo-labels | 200K pairs × pointmap | 25GB pointmaps, 30 GPU-h teacher forward | Generic 3D supervision |
| CLEVR-3D (synthetic) | 50K spatial QA | <1GB | Spatial relations |

### Architecture extensions (+25M params on 375M base)

```
view_a image bytes ─▶ UnifiedEmbed (existing) ─┐
view_b image bytes ─▶ UnifiedEmbed (existing) ─┼─▶ HybridBlock × 14 ─▶ outputs
DUSt3R pointmap ────▶ EGNN adapter (8M, 4 lyr) ─┤
camera params ──────▶ Plücker MLP (2M)        ─┘
```

Special tokens: `<view_a>`, `<view_b>`, `<pointmap>`, `<scene_3d>`.

### Loss (multitask, single stage)

```
L = 0.4 · L_lm                  # next-token across all modalities
  + 0.3 · L_pointmap_recon      # MSE on DUSt3R-teacher pointmap (confidence-weighted)
  + 0.2 · L_view_consistency    # InfoNCE on <scene_3d> token across same scene's 2 views
  + 0.1 · L_qa                  # CE on CLEVR-3D + ScanQA train
```

### Schedule (140 GPU-h on A100×2)

| Phase | Hours | What |
|-------|-------|------|
| 0 | 10 | Freeze base; train EGNN adapter + Plücker MLP only. Observe PLIF spike rate. |
| 1 | 60 | Unfreeze last 4 CfC blocks + adapter. lr=2e-4 cosine, bs=32, seq=2048 |
| 2 | 50 | Full unfreeze. lr=5e-5. Add CLEVR-3D + ScanQA in mix. |
| 3 | 20 | SFT on ScanQA train + 3DSRBench train, lr=1e-5 |

### Eval gates (rollback if missed)

- **G1 (post-Phase 1)**: pointmap MSE < 0.15 on Habitat val (DUSt3R teacher = 0.08; we accept 2× degradation)
- **G2 (post-Phase 2)**: CLEVR-3D spatial-relation accuracy ≥ 65% (random=25%, GPT-4V≈82%)
- **G3 (post-Phase 3)**: ScanQA val EM ≥ 18% (3D-LLM 2307.12981 reports 20.4% with 3B params)
- **G4 (always)**: WikiText ppl regression < 8% vs pre-3D base

## Why it beats alternatives

- **vs train-3D-encoder-from-scratch**: saves 500+ GPU-h. DUSt3R already encodes the
  geometric prior; we only learn the language↔geometry binding.
- **vs NeRF / Gaussian Splatting**: those are scene-fitting representations, not
  generalizable. Pointmap from DUSt3R is feedforward, scales linearly with samples.
- **vs voxels / mesh**: pointcloud already supported by UnifiedEmbed — zero
  engineering cost.
- **vs Zero-1-to-3 single-image**: 2-view triangulation is metrically correct;
  single-image is hallucinated.
- **vs ScanNet-only**: 1500 scans saturate a 375M model in <10K steps. Habitat
  infinite + Objaverse + ImageNet pseudo-labels gives 3+ orders of magnitude more
  variety at zero cost.

## LNN+SNN-specific risks

1. **PLIF spike-rate collapse on pointmap regression** — continuous 3D coords have
   no natural quantization; membrane potentials drift.
   **Mitigation**: pointmap branch goes through dense CfC only (PLIF observe-only)
   for first 30 GPU-h. Activate spikes when spike rate > 5%.

2. **CfC version-mismatch from EGNN coordinate updates** — EGNN mutates `x_i` in
   place; if shared with CfC fast-weights, autograd breaks
   (`feedback_torch_buffer_inplace.md`).
   **Mitigation**: emit detached `x_clone` to CfC, mutate original.

3. **STDP retrieval head pulls toward 2D image priors** — with 80% rendered/pseudo
   data, STDP's Hebbian co-firing memorizes view-pair surface stats, not 3D structure.
   **Mitigation**: STDP pause on pointmap loss steps; only update on L_qa + L_lm.
   Add view-augmentation (random rotation 0-360° around vertical) so STDP cannot
   exploit canonical pose.

## Anti-fakery test

Zero out the image embedding in `view_a` → caption/QA quality must collapse to
<50%. If model still answers correctly, it's text-only-cheating via dataset bias.
Fail = abort.
